The problem

Checkout payment options are not always one-size-fits-all

WooCommerce stores often need different payment methods for different countries, shipping destinations, currencies, or order values. Managing those decisions manually or with custom code is fragile.

The solution

Rules for payment gateway visibility

Payment Gateway Rules for WooCommerce lets admins create show or hide rules for installed WooCommerce gateways. Payment processing remains handled by Stripe, PayPal, Square, WooPayments, bank transfer, COD, or whichever gateways your store already uses.

Rule examples

Hide COD outside Australia

Use a shipping country rule to hide Cash on Delivery for international orders.

Show Square for AU orders

Create a billing or shipping country rule for AU and show only the selected local gateway.

Hide bank transfer below $100

Use a cart total rule with a less-than condition to hide manual payment options for small orders.

Show gateways by currency

Protect checkout

WooCommerce product image SEO

How to optimise WooCommerce product images for SEO

WooCommerce product image optimisation starts with clear product visuals, useful metadata, and a workflow that lets you review changes before publishing.

Why product image SEO matters

Product images help shoppers understand size, colour, material, compatibility, and quality before they read the full description. Search engines also use image metadata, surrounding product data, page context, and accessibility signals to understand what each image represents.

Use descriptive filenames

Replace camera exports and editing-app filenames with short, product-aware names. A poor filename is photoroom-20251212-154642-scaled.webp. A better filename is hi-capa-holster-gel-blaster-accessory.webp.

Write helpful alt text

Good alt text describes the product naturally. For example: Hi Capa holster for gel blaster pistols shown as a black tactical sidearm accessory.

Keep backgrounds and dimensions consistent

Consistent backgrounds, framing, and image dimensions make product grids easier to scan. Use compression and modern image formats without losing important product detail.

Create a review workflow

Before updating WooCommerce media, compare the original and processed image, review SEO metadata, and approve only the images that are ready for your store.

Alt text

How to write alt text for WooCommerce product images

Good WooCommerce alt text describes the product in plain language and supports accessibility without keyword stuffing.

Describe the product, not the file

Use the product name, type, colour, material, compatibility, or category where useful. Avoid camera export names, dates, or repeated keyword lists.

Use natural category context

For example, "Black Hi Capa holster for gel blaster pistols, shown from the front with tactical belt clip detail" is more useful than "image of product".

Featured images and gallery images

Featured image alt text should usually describe the product overall. Gallery image alt text can describe a specific angle, included accessory, close-up detail, or compatibility feature.

Common mistakes

  • Repeating the filename.
  • Using only the product SKU.
  • Stuffing the same keyword into every image.
  • Leaving alt text blank for important product images.

Background replacement

How to replace product image backgrounds in WooCommerce

Replacing product image backgrounds can make a WooCommerce catalogue feel more consistent, but the workflow should protect original images.

Choose a background style

White and grey backgrounds are useful for clarity. Brand backgrounds can help create a stronger store identity when used consistently.

Keep review controls

Do not automatically replace every image without checking the result. Review shadows, lighting, framing, and product edges before approval.

Preserve originals

A safe WooCommerce workflow imports the processed image, updates the selected image slot, and keeps the original attachment available for rollback.

Watch for common mistakes

Heavy shadows, clipped product edges, warped product shapes, and inconsistent background styles can make a catalogue look less trustworthy. Review before approving bulk updates.

What Optivra Image Studio does

Optivra connects your WooCommerce store to an image intelligence workflow built for ecommerce catalogue work.

Scan

Find featured, gallery, variation, and supported category images already in WooCommerce.

Report

Generate a Product Image Health Report with scores, insights and recommendations.

Queue

Select the images you want to process and keep the rest untouched.

Process

Send selected images to Optivra for AI background replacement and product image optimisation.

Review

Compare before and after, edit SEO metadata, approve, reject, reprocess, or revert.

The plugin does not automatically replace images unless you approve them or enable an auto-approval setting.

Requirements before you start

  • WordPress admin access
  • WooCommerce installed and active
  • An Optivra account and Site API Token
  • Available image processing credits
  • Product images uploaded to WooCommerce
  • Supported images: JPG, JPEG, PNG, and WebP

Installing the WooCommerce plugin

  1. Download the plugin zip from your Optivra account or download area.
  2. In WordPress Admin, go to Plugins → Add New → Upload Plugin.
  3. Upload the plugin zip and click Install Now.
  4. Click Activate.
  5. Open the Optivra Image Studio menu in WordPress admin.

WordPress.org review is pending. Download the plugin ZIP directly from Optivra and install it manually.

Download Plugin

If WooCommerce is inactive, activate WooCommerce first. The plugin needs WooCommerce products and media to scan your catalogue.

How to connect Optivra to WooCommerce

  1. Log into the Optivra website.
  2. Open your account or connected sites area and copy the Site API Token.
  3. In WordPress, open Optivra → Settings.
  4. Paste the token, then click Connect or Test Connection.
  5. Confirm the connected domain, plan, credits, and reset date.

After connection, Optivra Image Studio records operational service data such as plugin version, WordPress/WooCommerce/PHP versions, scan counts, queue counts, processing status, credit balance, and error diagnostics. This helps provide account status, credits, processing, and support without collecting orders, payment details, API tokens, passwords, secret keys, or WordPress user lists.

If the token fails, check for copied spaces, an inactive account, a wrong store token, SSL issues, or a temporary API connection problem.

Configuring image settings

For a first pass, keep review required before replacement enabled and test with a small product category.

  • Review requiredRecommended. Prevents changes before you approve them.
  • Include featured, gallery, and category imagesChoose which image slots are discovered and queued.
  • Auto-process scanned imagesLeave off until you trust your output style.
  • Duplicate detectionHelps avoid processing the same image repeatedly.
  • Smart product framingKeeps products large and safely framed.
  • Batch size and retry failed jobsControl how many images run at once and how failures are handled.
  • Low credit pausePrevents large batches from starting when credits are nearly depleted.

Background settings

Use a preset background for a clean, consistent catalogue style, or upload a custom background when your brand needs a specific look.

Category-specific background presets were intentionally removed from Settings to keep the workflow simple. Use global defaults and review individual results before approval.

Shadow and lighting settings

Off

No generated shadow.

Under product

Creates a subtle contact shadow below the item.

Behind product

Adds depth behind the product using a softer silhouette.

Custom

Adjust opacity, blur, offset, spread, softness, and colour.

Lighting enhancement can run automatically or target the product/whole image. Use brightness, contrast, highlight recovery, shadow lift, and neutralise tint gently.

Use subtle shadows and lighting corrections for ecommerce. Heavy effects can make products look artificial.

How to scan WooCommerce product images

  1. Open Optivra → Scan Products.
  2. Choose all products or a selected category.
  3. Select product status, product type, stock status, and image slots.
  4. Click Scan Product Images.
  5. Review found images and add selected images to the queue.

Scanning does not use credits. Credits are used only when images are processed.

Image framing and edge controls

Image to edge controls are per image/job, not global settings. Use them only where a product needs to touch one or more sides of the final image.

  • Bottom edgeUseful for shoes, bottles, or products that should sit on the lower frame.
  • Left or right edgeUseful for long horizontal products that should align to one side.
  • Top edgeUseful for tall products where the top should anchor to the frame.
  • All edgesUse only when the product should fill the frame.

How to review and approve AI product images

  1. Open Review & Approve.
  2. Compare the Before and After previews.
  3. Open the larger preview when you need more detail.
  4. Check the product name, image slot, and SEO metadata fields.
  5. Edit metadata if needed.
  6. Approve one image, approve selected images, reprocess, reject, or revert.

Do not bulk approve images until you have checked the output style and SEO metadata.

How to generate SEO image metadata for WooCommerce

Good image metadata helps accessibility, filenames, image titles, captions, and search engines understand the product and category context.

Filename

Clean, lowercase, hyphenated, product-specific.

Alt text

Natural description for accessibility and context.

Title

Concise product-focused attachment title.

Caption

Optional short customer-facing image context.

Description

Richer attachment description using product/category detail.

Example: Hi Capa Holster

Filename: hi-capa-holster-gel-blaster-accessory.webp

Alt: Hi Capa holster for gel blaster pistols, shown as a black tactical sidearm accessory.

Title: Hi Capa Holster - Gel Blaster Accessory

Caption: Black Hi Capa holster for gel blaster pistol loadouts and tactical setups.

Description: Optimised product image for the Hi Capa Holster, a tactical gel blaster accessory available from Azraels Armoury.

Avoid old export names like photoroom-20251212-154642-scaled. Regenerate or edit SEO fields before approval.

Approving and replacing images

On approval, the processed image is imported into the WordPress Media Library, SEO metadata is applied, and the correct featured, gallery, or supported category image slot is replaced.

Original images should be preserved for rollback/history. Approve a small batch first before approving larger runs.

Troubleshooting

After image is not showing

Refresh the page, reprocess the image, or contact support with the job ID. The preview URL may be expired, missing, or not synced yet.

“Not Found” when opening image

The processed URL may be wrong, expired, or missing metadata. Reprocess the job or provide the job ID to support.

Settings do not save

Your WordPress session may have expired, permissions may be insufficient, or the plugin may need updating. Reload the page and try again.

Token invalid

Check copied spaces, account status, store domain, and whether the token belongs to this site.

No images found after scanning

Products may have no images, filters may be too narrow, or the selected image slots may be disabled.

Credits not updating

Stripe checkout may still be syncing, payment may be incomplete, or the page may be cached.

Buy Credits button does nothing

Log in first. If it still fails, billing may need setup or the route may be returning an error.

SEO fields look like old filenames

Regenerate SEO metadata and check that the product has a title, category, tags, or attributes.

Best practices

  • Start with 5–10 products.
  • Review before approving.
  • Use consistent background and lighting settings.
  • Avoid over-processing or heavy shadows.
  • Keep filenames natural and product-specific.
  • Use category context in alt text.
  • Process high-value products first.
  • Monitor credits before large batches.

FAQ

Does scanning use credits?

No. Scanning only finds images. Credits are used when images are processed.

Will Optivra replace images automatically?

Only if configured to do so. The recommended workflow is review before approval.

Can I edit SEO fields before approval?

Yes. Metadata can be reviewed and edited before final approval.

Are original images deleted?

Original images should be preserved unless a specific setting says otherwise.

Can I reprocess an image?

Yes. Failed or unsatisfactory images can be reprocessed.

Why is my processed image not displaying?

The image may need a refreshed preview URL or the processed metadata may not have synced.

Can agencies use this for multiple stores?

Yes, if the Optivra account or plan supports multiple stores.

Is Stripe handled in WordPress?

No. Billing is handled securely through the Optivra website and Stripe. The plugin only shows billing and credit status.

Payment Gateway Rules for WooCommerce Guide

Learn how to install the plugin, create gateway rules, test checkout behaviour, and troubleshoot common issues.

What the plugin does

Payment Gateway Rules for WooCommerce controls which installed WooCommerce payment gateways appear at checkout based on rules. It does not replace Stripe, PayPal, Square, WooPayments, bank transfer, cash on delivery, or any other payment gateway.

The plugin does not process payments and does not store card details. Payment processing remains handled by your enabled WooCommerce gateways.

Requirements

  • WordPress 6.3 or later.
  • WooCommerce 8.0 or later installed and active.
  • PHP 7.4 or later.
  • At least one active WooCommerce payment gateway.
  • Administrator access to WordPress.
  • HTTPS is strongly recommended for checkout.

Installation

  1. Download the plugin ZIP.
  2. Open WordPress Admin → Plugins → Add New.
  3. Click Upload Plugin.
  4. Choose the ZIP file and click Install Now.
  5. Click Activate.
  6. Make sure WooCommerce is active.
  7. Open WooCommerce → Gateway Rules.
Download ZIP

Where to find settings

The plugin adds a submenu at WooCommerce → Gateway Rules. This page contains rule management, safety fallback settings, and the debug/test panel.

Creating your first rule

  1. Open WooCommerce → Gateway Rules.
  2. Enter a clear rule name.
  3. Enable the rule.
  4. Choose a match type: billing country, shipping country, active currency, or cart total.
  5. Enter the relevant country code, currency code, or cart total condition.
  6. Select one or more target gateways.
  7. Choose whether to show only selected gateways or hide selected gateways.
  8. Set a priority number and save.
  9. Test checkout before relying on the rule.

Common rule examples

Hide Cash on Delivery outside Australia

Use a shipping country rule and hide COD when the destination is not AU.

Hide bank transfer below $100

Use a cart total less-than rule and hide bank transfer for smaller carts.

Show local gateways for selected countries

Use billing or shipping country rules and show only selected local payment methods.

Control gateways by currency

Use the active currency rule when WooCommerce reports the correct currency code.

User role, product/category, and shipping method rules are not included in version 1.0.0. Do not plan workflows around those conditions unless a later version adds them.

Testing rules

  • Use an incognito/private browser window.
  • Test guest checkout and logged-in checkout where relevant.
  • Try different billing countries, shipping countries, currencies, and cart totals.
  • Clear caches and avoid caching checkout pages.
  • Check the debug/test panel for detected context, matched rules, and final visible gateways.

Troubleshooting

Gateway not hiding

Check that the rule is enabled, the correct gateway is selected, and the condition matches the current checkout context.

Gateway not showing

Confirm the gateway is enabled in WooCommerce and that a higher-priority rule is not hiding it later.

Shipping country rules do not apply

Some checkout states may not have a shipping country yet. Test with a full address and shipping destination entered.

Checkout Blocks issue

The plugin uses WooCommerce gateway filtering. Classic checkout is the primary supported flow; test Checkout Blocks carefully before relying on production rules.

No payment methods available

Keep the default safety fallback enabled unless you intentionally want checkout to show no methods when all gateways are hidden.

Cache issue

Do not cache checkout, cart, or checkout fragments. Clear page and object caches after changing rules.

FAQ

Does this plugin process payments?

No. It controls gateway visibility. Payments are still processed by your existing WooCommerce payment gateways.

Does it store card details?

No. It does not store card details or payment credentials.

Can I hide Cash on Delivery for international customers?

Yes. Use billing country or shipping country rules with your COD gateway selected.

Can I create multiple rules?

Yes. Rules can be enabled, ordered by priority, and targeted to one or more gateways.

Does it work with WooCommerce Checkout Blocks?

Checkout Blocks support should be tested on your store. The plugin primarily hooks WooCommerce's available payment gateways filter.

What happens if all gateways are hidden?

By default, the plugin restores all gateways to protect checkout. You can change this safety behaviour in settings.
